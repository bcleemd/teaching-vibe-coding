# 6부: 터미널을 넘어 스마트폰으로 - 텔레그램 챗봇 연동 영어 학습 비서 만들기

우리는 4부에서 모던 웹 영어 학습 앱을 만들었고, 5부에서 파일 백업을 자동화해 보았습니다.  
이번 6부에서는 한 걸음 더 나아가, 컴퓨터 앞에 앉아 있지 않아도 **스마트폰 텔레그램(Telegram) 메신저를 통해 매일 영어 퀴즈를 풀고 피드백을 받을 수 있는 '개인 영어 학습 챗봇'**을 구축합니다.

이 교안에서는 텔레그램 봇을 생성하는 기본 개념부터 시작하여, AI 에이전트(`agy CLI`)에게 챗봇 프로그램을 직접 구현하도록 지시하기 위한 **정밀 기획용 프롬프트 작성법**을 다룹니다.

---

## 1. 텔레그램 봇(Bot)의 특징과 장점

*   **언제 어디서나 학습**: PC의 터미널을 켜지 않고도 등하교, 출퇴근 시간에 스마트폰 메신저로 간편하게 퀴즈를 풀 수 있습니다.
*   **쉬운 외부 연동**: 카카오톡 등 다른 메신저에 비해 텔레그램은 개발자용 API가 무료로 완벽하게 열려 있어, API 키 발급과 봇 생성이 1분 만에 끝납니다.
*   **서버 제어의 기초**: 챗봇은 사용자의 메시지(입력)에 맞춰 서버가 데이터를 처리하고 응답을 돌려주는 백엔드 서버-클라이언트 아키텍처를 이해하는 가장 좋은 교재입니다.

---

## 2. 챗봇 개발 환경 언어 선택: Python vs Node.js

4부 웹 앱 실습에서는 브라우저 동작을 위해 **JavaScript**를 필수적으로 사용했지만, 백엔드 서버(대기 프로세스)로 구동되는 텔레그램 봇은 **파이썬(Python)**과 **Node.js(JavaScript)** 모두로 구현이 가능합니다. 이 가이드에서는 5부의 자동화 스크립트 흐름을 이어받아 **파이썬**을 기본 예제로 설명하되, 자바스크립트로 일관성 있게 구성하고 싶은 경우 Node.js를 대안으로 선택할 수 있습니다.

### 🐍 Python으로 구현할 때 (본 가이드 기본 채택)
*   **장점**:
    *   **코드의 직관성**: 문법이 간결하여 입문자가 코드를 읽고 맥락을 빠르게 파악하기가 매우 수월합니다.
    *   **자동화 연동의 편의성**: 5부에서 다룬 OS별 스케줄러(cron, schtasks) 및 시스템 관리 도구들과 매끄럽게 호환됩니다.
    *   **AI 및 데이터 분석 확장성**: 향후 챗봇에 학습 이력 데이터 분석이나 자연어 처리(NLP), 머신러닝 요소를 결합하기가 용이합니다.
*   **단점**:
    *   **언어 전환 비용**: 4부 웹 앱(JavaScript)과 다른 문법 체계(Python)를 새로 익혀 적용해야 하므로 일시적인 인지 부하가 생길 수 있습니다.

### 🟢 Node.js (JavaScript)로 구현할 때
*   **장점**:
    *   **단일 언어 스택**: 4부 웹 앱(`app.js`)과 6부 챗봇(`telegram_bot.js`)을 모두 JavaScript 하나로 개발할 수 있어 학습 집중도를 극대화할 수 있습니다.
    *   **자연스러운 JSON 핸들링**: 챗봇의 DB 역할을 하는 JSON(`sentences.json`, `progress.json`) 데이터 파일을 변환 작업 없이 바로 불러와 조작할 수 있습니다.
    *   **이벤트 중심 비동기 처리**: 실시간 채팅 메시지 수신 등 동시다발적인 요청에 Node.js 특유의 이벤트 루프 구조가 최적화되어 있습니다.
*   **단점**:
    *   **설정 파일 복잡도**: 단 하나의 파일로 실행되는 파이썬에 비해 `package.json` 및 `node_modules` 등의 개발 의존성 폴더 구조가 형성되어 파일 관리가 조금 더 무거워질 수 있습니다.

> 💡 **Node.js로 구현하고 싶다면?**  
> 만약 Node.js 환경으로 개발하고 싶다면, 아래 4번 항목의 프롬프트에서 `python-telegram-bot` 대신 **"Node.js 환경에서 `node-telegram-bot-api` 또는 `telegraf` 라이브러리를 사용해 telegram_bot.js를 작성해줘"**라고 에이전트(`agy CLI`)에게 지시를 변경하면 됩니다.

## 3. 텔레그램 봇 생성 및 준비 작업

텔레그램 봇을 작동시키기 위해서는 고유한 비밀번호 격인 **'봇 토큰(Bot Token)'**이 필요합니다.

1. **BotFather 검색**: 스마트폰 또는 PC의 텔레그램 앱에서 **`@BotFather`**(파란색 인증 마크가 있는 공식 계정)를 검색하여 대화를 시작합니다.
2. **봇 생성 명령 입력**: 대화창에 `/newbot`을 입력하고 전송합니다.
3. **이름 설정**: 봇의 이름(예: `내 영어 비서`)을 입력합니다.
4. **유저네임(ID) 설정**: 봇의 고유 ID를 입력합니다. 반드시 끝이 `_bot`으로 끝나야 합니다 (예: `vibe_english_study_bot`).
5. **토큰 복사**: 생성이 완료되면 `HTTP API token`이 발급됩니다. 이 토큰값(예: `123456789:ABCdefGhIJK...`)을 메모장에 안전하게 복사해 둡니다. (이 토큰은 절대 외부에 노출되면 안 됩니다!)

## 4. 에이전트(`agy CLI`)에 입력할 자동 구현 프롬프트

1부와 3부에서 다룬 `agy` 에이전트에게 챗봇 파일 작성을 지시하기 위한 상세 설계용 프롬프트입니다. 아래 프롬프트를 에이전트에 입력하여 파이썬 코드를 자동 생성합니다.

### 📝 AI 에이전트 지시용 프롬프트
```text
너는 파이썬(Python) 백엔드 개발자이자 초보자를 돕는 시니어 코더야.
우리가 이전에 만든 영어 학습 앱의 데이터(sentences.json)를 기반으로 작동하는 '텔레그램 영어 학습 챗봇'을 작성하려고 해.

아래 요구사항에 맞추어 `telegram_bot.py` 파일을 구현해줘.

[요구사항]
1. 라이브러리: `python-telegram-bot` (v20 이상 최신 버전) 라이브러리를 사용하여 비동기(async/await) 방식으로 파이썬 코드(telegram_bot.py)를 작성해줘.
2. 데이터 연동:
   - 기존의 `session4_english_app/sentences.json` 파일을 읽어와 문제 은행으로 사용해줘.
   - 사용자의 진행 상황(마지막으로 푼 문제 ID, 맞은 문장 ID 리스트, 틀린 문장 ID 리스트, 오늘 테스트 횟수 등)은 `progress.json` 파일에 저장하고 자동 업데이트해줘.
3. 명령어 및 기능:
   - `/start`: 봇 소개 및 사용법, 명령어 리스트 안내.
   - `/test`: 이전에 마지막으로 풀었던 문장 다음 번호부터 영어 테스트를 계속 진행함. 한 번 실행 시 최대 10개의 질문을 차례대로 물어봄. 
     - 봇이 한글 뜻을 메시지로 보내면, 사용자가 영어 문장을 입력함.
     - 사용자가 답을 보내면 봇이 대소문자 및 양끝 공백, 구두점(. , ? !)을 무시하고 정답 여부를 체크하여 알림.
     - 정답 여부를 `progress.json`에 즉각 반영한 뒤 다음 문제를 출제함. 10문제를 모두 풀거나 전체 문제를 완료하면 격려 메시지와 함께 종료.
   - `/review`: 지금까지 오답으로 기록된(틀린 문장 ID 리스트) 문장들만 필터링하여 복습 테스트를 진행함.
   - `/status`: 현재 전체 학습 진도율(예: 15 / 50 완료) 및 정답률(%) 통계를 내어 메시지로 전송함.
4. 보안 및 예외 처리:
   - 텔레그램 봇 토큰은 코드에 직접 하드코딩하지 않고, 환경 변수(`TELEGRAM_BOT_TOKEN`) 또는 `.env` 파일에서 로드하여 사용하도록 코드를 격리해줘.
   - `sentences.json`이 없거나 `progress.json` 파일이 손상되었을 때의 오류 예외 처리를 견고하게 추가해줘.
```

## 5. 챗봇 실행 및 구동 실습

에이전트가 코드를 완성했다면, 아래 단계를 통해 챗봇을 실행합니다.

### Step 1: 필요한 라이브러리 설치
터미널을 열고 파이썬 텔레그램 봇 라이브러리와 환경 변수 로드용 라이브러리를 설치합니다.
- **`uv` 패키지 매니저 환경**:
  ```bash
  uv add python-telegram-bot python-dotenv
  ```
- **기본 pip 설치 환경**:
  ```bash
  pip install python-telegram-bot python-dotenv
  ```

### Step 2: `.env` 환경 변수 설정
프로젝트 폴더 내에 `.env` 파일을 만들고 발급받은 토큰을 저장합니다.
```text
TELEGRAM_BOT_TOKEN=여기에_발급받은_텔레그램_토큰을_넣으세요
```

### Step 3: 챗봇 구동
터미널에서 작성된 챗봇 파일을 실행합니다.
```bash
python telegram_bot.py
```
*(실행 후 터미널에 에러 메시지가 없고 대기 상태를 유지한다면 챗봇이 켜진 것입니다.)*

### Step 4: 텔레그램 앱에서 연동 확인
1. 텔레그램 앱에서 내가 만든 봇 ID(예: `@vibe_english_study_bot`)를 검색합니다.
2. 대화창을 열고 `/start`를 누른 뒤, `/test`와 `/review` 명령어를 직접 쳐보며 대화형 영어 공부를 시작합니다!

## 6. 5부(스케줄러)와의 확장 연계 아이디어
5부에서 배운 **작업 스케줄러(cron 또는 schtasks)**를 활용하면 다음과 같은 고도화가 가능합니다.
*   **아침 배달 알림**: 매일 아침 8시가 되면, 챗봇이 먼저 사용자에게 텔레그램 메시지로 `"오늘의 영어 테스트를 시작해 보세요! /test 입력"` 이라는 모닝 알림 메시지를 보내도록 스케줄러와 파이썬 스크립트를 배치할 수 있습니다.
*   이러한 기능 연계를 통해 컴퓨터 속 터미널을 넘어 실제 나만을 위한 인공지능 개인 비서 시스템을 확장해 나가는 흐름을 구축할 수 있습니다.
